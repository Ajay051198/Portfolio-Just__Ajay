Car Health Monitor

In this project I (in a team of four) designed a tunable car health monitoring system which notifies the user when the car needs maintenance along with the predicted maintenance cost via an email along with a a simulator of the cars sensors to present and test the application.


The sub tasks included:

  1. Collect data about sensors values and reasons for car maintenance to decide checking the parameters and generate data to train predictive model.
  2. Design GUI for the simulator of the car’s sensors which streams data packets of sensor information.
  3. Setup AMQP for streaming this data.
  4. Design a GUI that displays the sensor value in a graphical form.
  5. Build monitoring system which decodes the packets of sensor information and checks if the car requires maintenance based on the checking parameters set by a .txt file and passes the sensor information to the predictor system.
  6. Create a predictive model (Regression model) that uses multiple features such as the cars make, the cars model and the sensors output (that is received from the monitoring system) to estimate the maintenance cost.
  7. Create a Notification layer that uses Google Mail API to extract all the previously collect information and notify the user via email.

Final say:

This project was completed succesfully under the supervision of Christoph Sielmann. It was awarded a grade of 83/100


Smart Blinds

Having the blinds cover the windows at night can save a lot of heat while having them open during the day while the sun is bright, can allow your house to be naturally heated. Also, the blinds being closed during the day or open at night can also be a good indication to burglars that you are away from your house.


To tackle these simple problems, me and my friend proposed and built a smart blind system which would keep the heat inside the house and lower the chances of a robbery, We also decided to add an Alexa integrated voice activation system which would turn a simple blind into a smart home system appliance that can be operated from anywhere in the house.


This was done with the help of an MSP430 board which controls a stepper motor that would move the rollers up or down automatically by using Alexa voice commands and also sample the light sensor (photoresistor) to incorporate a day-night mode where the roller shade moves up or down depending on the ambient light level, and finally a Raspberry Pi board, which is used to receive voice commands via a microphone.


Final say:

This project seemed direct and easy as first, however me and my peer struggled incorporating a motor strong enough, but had low power-consumption and heat dissipation, however we managed to pull it together and the project was completed successfully.


Swarming behaviour in the Robocone ecosystem

In this particular project I dealt with creating a deployment system for the existing nodes of the Robocone Project. This involved conducting simulations in MATLAB and testing the designed algorithms on the University of Manchester’s own MONA robots that were coded in Arduino.


The Robocone ecosystem is an ongoing project that focuses on building an autonomous robotic traffic cone system to reduce the number of accidents while also reducing the number of resources required. My task was to implement swarming behaviour in the deployment task with most accuracy.


The end result was a group of robots that were able to navigate a mission and cone off a pre-determined area via software.


This work was done as part of my final year project at Univeristy of Manchester.
